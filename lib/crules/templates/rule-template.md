# ルールテンプレート

## 概要

このテンプレートは、プロジェクトのルールを定義するための基本的な構造を提供します。

# 大原則
- **このルールズの意義**: このプルジェクトルールズは、単にAIへの指示という意味だけでなく、開発を進める上で今までの開発内容を確認するためのドキュメントという意味合いをもっています。

## ルールの基本構造

### 1. ルールの目的

- ルールが解決する問題
- 期待される効果
- 適用範囲

### 2. ルールの詳細

- 具体的な要件
- 制約条件
- 例外事項

### 3. 実装ガイドライン

- コード例
- ベストプラクティス
- 注意点

### 4. 検証方法

- レビューポイント
- テスト要件
- 自動化の可能性

### 5. 例外処理

- 例外が認められる場合
- 例外申請プロセス
- 代替案

## ルールの変更に関するガイド
- **ルール更新提案**: 開発を進めていく中で、ルールに反するような開発をおこなう可能性がでてきた場合は、および、ルールに新たな内容を付け加える必要がでてきた場合は、そのまま開発を進めないでください。この場合、ユーザーの合意を得てルールを変更してください。

## テンプレートの使用方法

1. このテンプレートをコピーして新しいルールファイルを作成
2. 各セクションを具体的な内容で更新
3. 必要に応じてセクションを追加・削除
4. 例や図を追加して理解を促進

## メンテナンス

- ルールの更新履歴
- 変更理由
- 影響範囲
